01. .at()

"문자열".at(위치값)

  • 지정한 숫자(index)를 받아, 문자열에서 행당되는 인덱스의 요소를 반환합니다.
  • 대소문자를 구별합니다.
  • 문열의 길이보다 큰 인덱스를 지정하면 undefiend를 반환합니다.
  • 음수인 경우 문자열의 오른쪽에서부터 인덱스를 셉니다.
{
        "javascript reference".at()       //j
        "javascript reference".at(0)      //j
        "javascript reference".at(1)      //a
        "javascript reference".at(2)      //v
        "javascript reference".at(3)      //a
        "javascript reference".at(4)      //s
        "javascript reference".at(5)      //c
        "javascript reference".at(-1)     //e
        "javascript reference".at(-2)     //c
        "javascript reference".at(-3)     //n
        "javascript reference".at(-4)     //e
        "javascript reference".at(-5)     //r
        "javascript reference".at(100)    // undefined
}

02. .charAt()

"문자열".charAt(위치값)

  • 문자열 내 특정 위치에 있는 문자를 반환하는 데 사용됩니다.
  • 지정된 인덱스에 있는 문자를 반환하며, 인덱스는 0부터 시작합니다.
  • 만약 지정된 인덱스가 문자열의 범위를 벗어난 경우, 빈 문자열을 반환합니다.
{
        const str = 'Hello, World!';
        console.log(str.charAt(0)); // 'H'
        console.log(str.charAt(7)); // 'W'
        console.log(str.charAt(20)); // ''
}

03. .charCodeAt()

"문자열".charCodeAt(위치값)

  • 문자열 내 특정 위치에 있는 문자의 유니코드(Unicode) 값을 반환합니다.
  • 지정된 인덱스에 위치한 문자의 유니코드 값을 나타내는 정수를 반환합니다.
  • 만약 지정된 인덱스가 문자열의 범위를 벗어난 경우, NaN을 반환합니다.
  • 문자열 처리에 있어 문자의 유니코드 값을 필요로 할 때 유용하게 사용됩니다.
{
        const str = 'Hello, World!';
        console.log(str.charCodeAt(0)); // 72
        console.log(str.charCodeAt(7)); // 87
        console.log(str.charCodeAt(20)); // NaN
}
  • 첫 번째 예제에서, 'H'의 유니코드 값인 72가 반환됩니다.
  • 두 번째 예제에서, 'W'의 유니코드 값인 87이 반환됩니다.
  • 세 번째 예제에서, 지정된 인덱스가 문자열의 범위를 벗어나므로 NaN이 반환됩니다.

04. .codePointAt()

"문자열".codePointAt(위치값)

  • 문자열 내 특정 위치에 있는 문자의 유니코드 코드 포인트 값을 반환합니다.
  • 이 메서드는 charCodeAt()와 유사하지만, 전체 유니코드 코드 스페이스 내의 모든 문자, 즉 기본 다국어 평면(BMP)을 넘어서는 문자까지도 정확하게 처리할 수 있습니다.
  • 특히 이모지나 특정 언어 문자와 같이 BMP 바깥의 문자를 다룰 때 중요합니다.
  • 지정된 인덱스가 문자열 범위를 벗어나는 경우 undefined를 반환합니다.
{
        const str = '👍🏽ABC';
        console.log(str.codePointAt(0)); // 128077, '👍'의 코드 포인트
        console.log(str.codePointAt(2)); // 127997, '🏽'의 코드 포인트
        console.log(str.codePointAt(3)); // 65, 'A'의 코드 포인트
}

05. .concat()

"문자열".concat(위치값)

  • 하나 이상의 문자열을 현재 문자열에 연결하여 새로운 문자열을 생성합니다.
  • 기존 문자열을 변경하지 않고, 결합된 새로운 문자열을 반환합니다.
  • 메서드는 매개변수로 전달된 모든 문자열을 순서대로 결합합니다.
  • 매개변수로 문자열이 아닌 값을 전달하면, 그 값이 문자열로 변환된 후에 연결됩니다.
{
        const str1 = 'Hello, ';
        const str2 = 'World';
        const str3 = '!';
        
        const result = str1.concat(str2, str3);
        console.log(result); // "Hello, World!"
}
  • 이 예제에서는 str1, str2, str3을 연결하여 "Hello, World!"라는 새로운 문자열을 생성합니다.

06. .includes()

"문자열".includes(위치값)

  • 특정 문자열이 다른 문자열 내에 포함되어 있는지 여부를 확인합니다.
  • 문자열이 포함되어 있으면 true를, 그렇지 않으면 false를 반환합니다.
  • 문열의 길이보다 큰 인덱스를 지정하면 undefiend를 반환합니다.
  • 대소문자를 구별합니다.
{
        let str = "javascript reference";

        console.log(str.includes("reference")); // true
        console.log(str.includes("JavaScript")); // false (대소문자 구분)
        console.log(str.includes("reference", 10)); // false (검색 시작 위치가 10이므로)
}

07. .indexOf()

"문자열".indexOf(찾고자하는 문자열, [위치값])

  • 찾고자 하는 문자열이 발견되면 해당 인덱스를 반환하고, 발견되지 않으면 -1을 반환합니다.
  • 문자열 내에서 특정 문자열의 첫 번째 인덱스를 반환합니다.
  • 특정 문자열이 포함되어 있는지 여부를 확인하는 데 사용됩니다.
  • 대소문자를 구별합니다.
{
        let str = "Hello, world!";

        console.log(str.indexOf("world")); // 7
        console.log(str.indexOf("hello")); // -1 (대소문자 구분)
        console.log(str.indexOf("l", 3)); // 3 (검색 시작 위치가 3이므로)
}

08. .lastIndexOf()

"문자열".lastIndexOf(찾고자하는 문자열, [위치값])

  • 문자열 안에서 주어진 값을 역순으로, 즉 문자열의 끝에서부터 시작하여 검색하고, 일치하는 첫 번째 위치의 인덱스를 반환합니다.
  • 대소문자를 구분하여 검색하고, 일치하는 값이 없으면 -1을 반환합니다.
  • 빈 문자열('')을 검색하는 경우, 위치값 + 1의 값을 반환합니다.
  • 검색을 시작할 위치의 기본값은 "문자열".length - 1로, 문자열의 끝에서부터 검색을 시작합니다. 만약 검색을 시작할 위치가 문자열의 길이보다 크거나 같으면, 전체 문자열이 검색 대상이 됩니다.
  • 검색을 시작할 위치가 음수인 경우는 0으로 취급됩니다.
{
        const str = 'Hello, World! Hello, World!';

        console.log(str.lastIndexOf('World')); // 25
        console.log(str.lastIndexOf('world')); // -1 (대소문자 구분)
        console.log(str.lastIndexOf('Hello', 10)); // 0
        console.log(str.lastIndexOf('')); // 27 (빈 문자열은 항상 문자열의 끝 인덱스를 반환)
}
  • 첫 번째 예제에서는 "World"가 문자열의 뒤쪽에서 처음으로 발견된 위치의 인덱스인 25를 반환합니다.
  • 두 번째 예제에서는 "world"를 찾지 못하므로 -1을 반환합니다.
  • 세 번째 예제에서는 fromIndex를 10으로 지정해서, "Hello"가 처음으로 발견된 위치의 인덱스인 0을 반환합니다.
  • 네 번째 예제에서는 빈 문자열을 검색했으므로 문자열의 길이, 즉 27을 반환합니다.

09. .localeCompare()

"비교의 기준 문자열".localeCompare(compareString[, locales[, options]])

  • compareString: referenceStr와 비교될 문자열입니다.
  • locales (선택적): 비교에 사용할 로케일의 문자열이나 그 배열입니다. 이 인수를 생략하면 기본 로케일이 사용됩니다.
  • options (선택적): 비교에 사용할 옵션을 지정하는 객체입니다. 이 객체를 통해 대소문자 구분, 정렬 순서 등을 세밀하게 조정할 수 있습니다.
  • 비교 문자열이 기준 문자열보다 앞에 오면 음수를 반환합니다.
  • 비교 문자열이 기준 문자열과 동일하면 0을 반환합니다.
  • 비교 문자열이 기준 문자열보다 뒤에 오면 양수를 반환합니다.
  • 현재 문자열과 비교할 문자열을 기준으로 정렬 순서를 수치로 반환합니다.
  • 다국어 문자열을 처리할 때 유용하며, 문자열의 비교는 로케일(언어 및 지역 설정)에 의존적입니다.
  • 이 메서드를 사용하면 문자열이 비교 대상 문자열보다 앞에 오는지, 뒤에 오는지, 또는 동일한지를 알 수 있습니다.
{
        const str1 = '사과';
        const str2 = '바나나';
        
        console.log(str1.localeCompare(str2)); // -1 (str1이 str2보다 앞에 옵니다)
        console.log(str2.localeCompare(str1)); // 1  (str2가 str1보다 뒤에 옵니다)
        console.log(str1.localeCompare(str1)); // 0  (두 문자열이 동일합니다)
}

옵션 객체의 속성 예시

  • sensitivity: 문자 비교의 민감도를 설정합니다. 예: 'base', 'accent', 'case', 'variant'
  • ignorePunctuation: 구두점을 무시할지 여부를 결정합니다.
  • numeric: 숫자를 문자열로 비교할지, 숫자로 비교할지 결정합니다.

10. .match()

"문자열".match(검색할 정규식 패턴)

  • 문자열에서 정규식과 일치하는 부분을 찾아 배열로 반환합니다.
  • 정규식과 일치하는 부분을 배열로 반환합니다. 일치하는 부분이 없으면 null을 반환합니다.
  • 정규식에 /g 플래그를 사용하면 모든 일치 항목을 반환합니다.
  • 배열의 첫 번째 요소는 전체 일치 문자열이며, 이후 요소는 캡처 그룹에 해당하는 부분 문자열입니다.
{
        let str = "The quick brown fox jumps over the lazy dog.";

        console.log(str.match(/quick/)); // ["quick"]
        console.log(str.match(/\w+/g)); // ["The", "quick", "brown", "fox", "jumps", "over", "the", "lazy", "dog"]
        console.log(str.match(/hello/)); // null
}

11. .matchAll()

"문자열".matchAll(검색할 정규식 패턴)

  • 문자열 내에서 정규 표현식과 일치하는 모든 결과를 찾아서, 이를 반복 가능한 iterator로 반환합니다.
  • 이 메서드는 정규식에 g 플래그(전역 검색)가 설정되어 있을 때 유용하며, 일치하는 각 결과에 대한 정보를 포함하는 객체를 반환합니다.
  • 만약 regexp가 정규 표현식이 아닌 경우, 내부적으로 new RegExp(regexp)를 통해 정규 표현식 객체로 변환됩니다.
  • matchAll() 메서드는 일치하는 모든 결과를 포함하는 iterator를 반환합니다.
  • 정규 표현식에 g 플래그가 없으면 TypeError가 발생합니다.
  • 일치하는 정보가 필요 없고, 단순히 일치하는 문자열만 필요한 경우 String.prototype.match() 메서드를 사용할 수 있습니다.

포함된 속성들

  • groups: 명명된 캡처 그룹을 포함하는 객체입니다. 캡처 그룹이 없는 경우, undefined가 될 수 있습니다.
  • index: 일치가 발생한 문자열 내의 시작 위치입니다.
  • input: 검색이 수행된 원본 문자열입니다.
  • 이외에도, 일치하는 문자열 자체가 결과 배열의 첫 번째 요소로 포함됩니다.
{
        const str = '2023년 5월 6일에 2024년 5월 6일';
        const regex = /\d{4}년 \d{1,2}월 \d{1,2}일/g;
        const matches = str.matchAll(regex);

        for (const match of matches) {
            console.log(match[0]);
            console.log(`Found at ${match.index}`);
        }
}

이 예제에서는 주어진 문자열 내에서 "YYYY년 MM월 DD일" 형식에 일치하는 모든 날짜를 찾아 출력합니다. matchAll() 메서드를 사용하여 반환된 iterator를 순회하면서, 각 일치하는 문자열과 그 위치를 출력합니다.

12. .normalize()

"문자열".normalize([form])

정규화의 형태를 지정하는 선택적 매개변수의 네가지 값

  • "NFC" (기본값): Normalization Form Canonical Composition. 여러 개의 문자를 조합하여 하나의 문자로 합치는 방식입니다.
  • "NFD": Normalization Form Canonical Decomposition. 하나의 문자를 여러 개의 조합 문자로 분해하는 방식입니다.
  • "NFKC": Normalization Form Compatibility Composition. 호환성을 고려하여 문자를 조합하는 방식입니다.
  • "NFKD": Normalization Form Compatibility Decomposition. 호환성을 고려하여 문자를 분해하는 방식입니다.
  • 유니코드 문자열을 정규화된 형태로 변환해주는 JavaScript의 내장 메서드입니다.
  • 유니코드 표준에 따라 문자열 내의 문자들을 동등한 형태로 표준화하여, 문자열 비교 및 처리를 용이하게 합니다.
  • 유니코드에는 동일하게 표현되어야 하는 문자가 서로 다른 바이트 코드를 가질 수 있는데, normalize() 메서드는 이러한 문제를 해결하는 데 유용합니다.
  • 문자열의 내용이 변경될 수 있습니다.
  • 정규화 과정에서 문자열의 길이가 변할 수 있습니다.
  • 일부 환경에서는 이 메서드가 지원되지 않을 수 있으므로, 사용 전에 호환성을 확인해야 합니다.
{
        const str = '\u1E9B\u0323';

        // NFC: 조합된 형태로 정규화
        console.log(str.normalize('NFC'));
        // NFD: 분해된 형태로 정규화
        console.log(str.normalize('NFD'));
        // NFKC: 호환성을 고려하여 조합된 형태로 정규화
        console.log(str.normalize('NFKC'));
        // NFKD: 호환성을 고려하여 분해된 형태로 정규화
        console.log(str.normalize('NFKD'));
}

13. .padEnd()

"문자열".padEnd(목표로 하는 문자열의 길이 [, "패딩에 사용될 문자열"])

  • 문자열을 특정 길이에 도달할 때까지 주어진 문자열로 끝에서부터 채워 새로운 문자열을 생성하고 반환합니다.
  • 목표로 하는 문자열의 길이와 현재 문자열의 길이가 이 값보다 크거나 같으면, 어떠한 패딩도 추가되지 않습니다.
  • 목표로 하는 문자열의 길이와 현재 문자열의 길이보다 작거나 같은 경우, 원본 문자열이 그대로 반환됩니다.
  • 패딩에 사용될 문자열을 생략하면 기본값으로 공백(" ")이 사용됩니다.
  • 만약 padString이 너무 길어서 targetLength를 초과하면, 필요한 길이만큼만 잘려서 사용됩니다.
  • 원본 문자열이 targetLength에 도달하거나 초과할 때까지 padString으로 끝에서부터 채워진 새로운 문자열입니다.
  • 원본 문자열을 변경하지 않습니다.
{
        const str = '\u1E9B\u0323';

        // NFC: 조합된 형태로 정규화
        console.log(str.normalize('NFC'));
        // NFD: 분해된 형태로 정규화
        console.log(str.normalize('NFD'));
        // NFKC: 호환성을 고려하여 조합된 형태로 정규화
        console.log(str.normalize('NFKC'));
        // NFKD: 호환성을 고려하여 분해된 형태로 정규화
        console.log(str.normalize('NFKD'));
}

14. .padStart()

"문자열".padStart(목표로 하는 문자열의 길이 [, "채움에 사용될 문자열"])

  • 주어진 길이에 도달할 때까지 현재 문자열 앞에 특정 문자열을 채워넣어 새로운 문자열을 반환합니다.
  • 특히 숫자나 문자열을 특정 형식에 맞춰 출력할 때 유용하게 사용됩니다.
  • 목표로 하는 문자열의 길이와 현재 문자열의 길이보다 크거나 같을 경우, 어떤 추가적인 채움도 발생하지 않습니다.
  • 목표로 하는 문자열의 길이와 현재 문자열의 길이보다 작거나 같다면, 어떠한 채움도 발생하지 않고 원본 문자열이 그대로 반환됩니다.
  • 패딩에 사용될 문자열을 생략하면 기본값으로 공백(" ")이 사용됩니다.
  • padString이 targetLength를 초과하여 필요한 경우, 필요한 길이만큼만 잘려서 사용됩니다.
  • 원본 문자열의 앞쪽을 padString으로 채워 길이를 targetLength에 맞춘 새로운 문자열입니다.
  • padString이 지정되지 않았거나 빈 문자열인 경우, 공백 문자가 채움 문자로 사용됩니다.
  • 원본 문자열을 변경하지 않습니다.
{
        const str = "42";

        console.log(str.padStart(5, "0"));    // "00042"
        console.log(str.padStart(10, "123")); // "1231231242"
        console.log(str.padStart(3));   
}

15. .repeat()

"문자열".repeat(count)

  • 문자열을 주어진 횟수만큼 반복하여 새로운 문자열로 만들고 반환하는 메서드입니다.
  • 문자열을 늘리고 싶을 때 유용하게 사용할 수 있습니다.
  • count가 0이면 빈 문자열("")이 반환됩니다.
  • count 값이 소수점인 경우 내림하여 정수로 처리됩니다.
  • 주어진 count만큼 반복된 새로운 문자열을 반환합니다. 원본 문자열은 변경되지 않습니다.
  • count 값이 음수거나, Infinity와 같이 매우 큰 수인 경우 RangeError가 발생합니다.
  • count 값이 소수일 경우, 소수점 이하는 무시되고 정수 부분만 사용됩니다.
  • 예를 들어, UI 요소에서 같은 문자나 문구를 반복해서 보여주어야 할 경우에 활용할 수 있습니다.
{
        const str = "abc";

        console.log(str.repeat(3)); // "abcabcabc"
        console.log(str.repeat(1)); // "abc"
        console.log(str.repeat(0)); // ""   
}

16. .replace()

"문자열".replace(regexp|substr, newSubstr|function)

  • regexp|substr: 교체할 대상이 되는 문자열 또는 정규식 패턴
  • newSubstr|function: 교체할 새로운 문자열 또는 교체 로직을 포함한 함수
  • 문자열에서 특정 패턴을 찾아 다른 문자열로 교체하여 새로운 문자열을 반환합니다.
  • 첫 번째 일치 항목만 교체하며, 정규식에 /g 플래그를 사용하면 모든 일치 항목을 교체할 수 있습니다.
  • 교체 문자열에 $1, $2 등의 특수 문자를 사용하면 캡처 그룹을 참조할 수 있습니다.
  • 교체 함수를 사용하면 더 복잡한 교체 로직을 구현할 수 있습니다.
{
        let str = "The quick brown fox jumps over the lazy dog.";
        
        console.log(str.replace("fox", "cat")); // "The quick brown cat jumps over the lazy dog."
        console.log(str.replace(/\w+/, "animal")); // "animal quick brown fox jumps over the lazy dog."
        console.log(str.replace(/\w+/g, (match) => match.toUpperCase())); // "THE QUICK BROWN FOX JUMPS OVER THE LAZY DOG."
}

17. .replaceAll()

"문자열".replaceAll("교체하려는 문자열", "교체할 새로운 문자열")

  • 문자열 내에서 주어진 패턴이나 문자열을 찾아 모두 다른 문자열로 교체한 후, 그 결과를 새로운 문자열로 반환합니다.
  • 문자열 내의 모든 인스턴스를 대체하려는 경우에 특히 유용합니다.
  • 교체하려는 문자열에 문자열 또는 정규 표현식(RegExp)을 사용할 수 있습니다. 문자열을 사용하는 경우, 대소문자를 구분하여 완전히 일치하는 모든 인스턴스를 찾습니다.
  • 교체할 새로운 문자열에는 함수를 사용할 수도 있으며, 이 경우 각 매치에 대해 함수가 호출되고, 함수의 반환값이 교체 문자열로 사용됩니다.
  • 원본 문자열을 변경하지 않습니다.
  • 정규 표현식을 사용하는 경우, 전역 플래그(g)를 사용하지 않으면 TypeError가 발생합니다.
{
        const phrase = 'The quick brown fox jumps over the lazy dog. If the dog reacted, was it really lazy?';

        console.log(phrase.replaceAll('dog', 'monkey'));
        // 출력: "The quick brown fox jumps over the lazy monkey. If the monkey reacted, was it really lazy?"
        
        // 정규 표현식을 사용하여 대소문자를 구분하지 않고 교체
        console.log(phrase.replaceAll(/dog/gi, 'monkey'));
        // 출력: "The quick brown fox jumps over the lazy monkey. If the monkey reacted, was it really lazy?"
}

18. .search()

"문자열".search("검색에 사용할 정규 표현식 객체")

  • 일치하는 문자열이 발견되면, 일치하는 부분의 시작 인덱스(0 이상의 정수)를 반환합니다.
  • 일치하는 문자열이 없으면 -1을 반환합니다.
  • 전역 검색 플래그(g)를 무시합니다. 즉, 정규 표현식에 전역 검색 플래그(g)가 있어도, 첫 번째 일치하는 결과만 찾고 검색을 중단합니다.
  • 호출된 문자열을 변경하지 않습니다. 즉, 이 메서드는 순수 함수(pure function)의 특성을 가집니다.
{
        var str = "Hello, world!";
        var n = str.search(/world/);
        console.log(n); // 7 출력
}

19. .slice()

"문자열".slice(추출 시작점의 인덱스[, 추출을 종료할 인덱스])

  • 두 개의 매개변수를 받음
  • 첫 번째 매개변수는 추출이 시작되는 인덱스를 지정하고, 두 번째 매개변수는 추출이 끝나는 인덱스를 지정합니다.
  • 두 번째 매개변수는 선택사항이며, 제공되지 않을 경우 문자열의 끝까지 추출합니다.
  • 추출을 종료할 인덱스에 해당하는 문자는 추출되지 않습니다. 
{
        var str = "Hello, world!";
        console.log(str.slice(7, 12)); // "world"
        console.log(str.slice(-6, -1)); // "world"
        console.log(str.slice(7)); // "world!"
        console.log(str.slice(-6)); // "world!"
}
  • 첫 번째 예제에서는 인덱스 7부터 12 전까지의 문자열인 "world"를 추출
  • 두 번째 예제에서는 문자열 끝에서부터 6번째 위치부터 끝에서 1번째 위치 전까지인 "world"를 추출
  • 세 번째 예제에서는 인덱스 7부터 문자열의 끝까지인 "world!"를 추출
  • 네 번째 예제에서는 문자열 끝에서부터 6번째 위치부터 문자열의 끝까지인 "world!"를 추출

20. .substring()

"문자열".substring(indexStart[, indexEnd])

    * indexStart: 추출을 시작할 0부터 시작하는 인덱스입니다. 만약 indexStart가 indexEnd보다 크면, substring은 두 매개변수를 자동으로 교환하여 처리합니다.
    * indexEnd: 추출을 종료할 인덱스로, 이 위치의 문자는 포함되지 않습니다. 이 매개변수는 선택적이며, 제공되지 않을 경우 문자열의 끝까지 추출됩니다. indexEnd가 문자열의 길이보다 큰 경우, 문자열의 끝이 추출의 종료점이 됩니다.
  • 두 개의 매개변수를 받을 수 있습니다.
  • 원본 문자열은 변경하지 않고, 지정된 부분만을 포함하는 새로운 문자열을 반환합니다.
  • 두 번째 매개변수는 선택사항이며, 제공되지 않을 경우 문자열의 끝까지 추출합니다.
  • 음수 인덱스를 허용하지 않습니다. 음수 값은 0으로 취급됩니다.
{
        var str = "Hello, world!";
        var result = str.substring(7, 12);
        console.log(result); // "world" 출력
}

21. .substr()

"문자열".substr(추출을 시작할 위치[, 추출할 문자의 수])

  • 문자열의 특정 부분을 추출하여 새로운 문자열을 반환합니다.
  • 두 개의 매개변수를 받습니다: 첫 번째 매개변수는 시작 위치를 나타내며, 두 번째 매개변수는 추출할 길이를 나타냅니다.
  • 추출을 시작할 위치에 음수가 주어지면, 문자열의 끝에서부터의 거리로 계산됩니다. 예를 들어, -3은 문자열 끝에서부터 세 번째 문자를 시작 위치로 합니다.
  • 추출할 문자의 수의 매개변수를 생략하면 start에서 시작하여 문자열의 끝까지 모든 문자가 추출됩니다.
  • 원본 문자열에서 지정된 부분 문자열을 포함하는 새 문자열입니다.
  • substr() 메서드는 ECMAScript 2015(ES6) 이후로는 권장되지 않으며, 향후 JavaScript 버전에서는 제거될 수 있습니다.
{
        let str = 'Mozilla';

        console.log(str.substr(1, 2));  // "oz"
        console.log(str.substr(2));     // "zilla"
        console.log(str.substr(-3));    // "lla"
        console.log(str.substr(-3, 2)); // "ll"
}

22. .split()

"문자열".split(["문자열을 분할할 구분자"[, 반환할 배열의 최대 길이]])

  • 두 개의 매개변수를 받을 수 있습니다.
  • 구분자는 문자열이나 정규 표현식(RegExp)이 될 수 있습니다.
  • 만약 구분자가 생략되거나 빈 문자열('')이 제공되면, 문자열의 각 글자가 분리되어 배열로 반환됩니다.
  • 여러 개의 부분 문자열로 나누고, 그 결과를 배열 형태로 반환합니다.

  • 예제 2번 : 공백 " "을 구분자로 사용하여 문자열을 단어 단위로 분할합니다.
  • 예제 3번 : "-"를 구분자로 사용하여 날짜 형식의 문자열을 연, 월, 일로 분할합니다.

23. .startsWith()

"문자열".startsWith("특정 문자열"[, 검색 시작 위치])

  • 두 개의 매개변수를 받을 수 있습니다.
  • 첫 번째 매개변수 "특정 문자열"은 시작하는 문자열이 해당 문자열로 시작하는지 확인하는 기준이 됩니다.
  • 두 번째 매개변수는 검색을 시작할 문자열의 위치를 지정합니다. 생략 가능하며, 기본값은 0입니다.
  • 메서드는 문자열이 특정 문자열로 시작하는지 확인하고, 그 결과를 불린(true, false)으로 반환합니다.
{

        //예제 2번

        var result = str.startsWith("world");
        console.log(result); // false

        //예제 3번

        console.log(result); // true

  • 예제 1번 : 문자열 "Hello, world!"가 "Hello"로 시작하기 때문에 true를 반환합니다.
  • 예제 2번 : 문자열 "Hello, world!"가 "world"로 시작하지 않기 때문에 false를 반환합니다.
  • 예제 3번 : 문자열 "Hello, world!"에서 7번째 위치부터 시작해 "world"로 시작하기 때문에 true를 반환합니다.

24. .endsWith()

"문자열".endsWith("검색할 문자열"[, "문자열의 길이"])

  • 첫 번째 매개변수는 검색할 문자열입니다.
  • 두 번째 매개변수(선택 사항)는 검색을 수행할 문자열의 길이입니다. 이 매개변수를 사용하면 문자열의 길이를 인위적으로 제한할 수 있습니다.
  • 문자열이 지정된 문자열로 끝나는지 여부에 따라 true 또는 false를 반환합니다.

}
  • 예제 1번 : 문자열 "Hello, world!"가 "world!"로 끝나므로 true를 반환합니다.
  • 예제 2번 : "JavaScript is fun" 문자열의 첫 17개 문자로 제한하였을 때, "fun"으로 끝나므로 true를 반환합니다.

30. .trim()

  • 문자열의 양 끝(앞과 뒤)에 있는 공백을 제거하고, 그 결과로 새로운 문자열을 반환합니다.
  • trimmedString : 원본 문자열의 앞과 뒤에서 공백이 제거된 새로운 문자열이 반환됩니다.
  • trim() 메서드는 문자열의 중간에 있는 공백을 제거하지 않습니다. 예를 들어, "Hello, world!"에서 trim()을 사용하면, 중간의 여러 공백은 그대로 유지됩니다.
  • 원본 문자열에 변경을 가하지 않으며, 공백이 제거된 새로운 문자열을 반환합니다.
{

        //trim() 메서드는 문자열의 중간에 있는 공백을 제거하지 않습니다. 
        //예를 들어, "Hello,    world!"에서 trim()을 사용하면, 중간의 여러 공백은 그대로 유지됩니다.
}
  • 이 예시에서, str.trim()은 원본 문자열 str의 앞과 뒤에 있는 공백을 제거하고, 공백이 제거된 새로운 문자열 "Hello, world!"를 반환합니다.

  • 메서드는 원본 문자열을 변경하지 않고, 공백이 제거된 새로운 문자열을 반환합니다. 여기서 공백에는 스페이스, 탭, 줄바꿈 문자 등이 포함됩니다.
